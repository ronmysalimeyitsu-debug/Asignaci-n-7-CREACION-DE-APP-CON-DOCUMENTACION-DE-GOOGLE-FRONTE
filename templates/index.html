Elementos

Inspecciona el DOM y el CSS en tiempo real aplicado al sistema de leyes venezolanas y códigos legales.

Consola

Documentación interactiva de la consola de desarrollo front-end:

  • console.log(): Utilizado para depurar el flujo general de la aplicación web y respuestas HTTP.
  • console.table(): Estructura los registros jurídicos devueltos por SQLite en formato tabular legible.
  • console.error(): Captura fallos de red asíncronos al consultar la API REST de Flask.

Fuentes

Documentación de la pestaña Fuentes (Sources): Permite depurar scripts JavaScript asociados en static/js/main.js, establecer breakpoints de ejecución paso a paso y evaluar la jerarquía de archivos estáticos del proyecto.

Red

Documentación de la pestaña Red (Network): Audita en tiempo real las peticiones HTTP asíncronas (Fetch/XHR) dirigidas hacia las rutas del backend en Flask, midiendo tiempos de latencia y la estructura JSON de la API: /api/buscar?q=...

Memoria

Documentación de la pestaña Memoria (Memory): Utilizada para analizar instantáneas de montones de memoria (Heap Snapshots) y detectar posibles pérdidas de memoria (memory leaks) durante consultas masivas de textos jurídicos en SQLite.

Solicitud

Documentación de la pestaña Aplicación / Almacenamiento (Application): Inspecciona el almacenamiento local (LocalStorage), cookies, bases de datos IndexedDB y sesiones persistidas en el navegador del cliente.

Seguridad

Documentación de la pestaña Seguridad (Security): Verifica certificados SSL/TLS, estado de conexiones HTTPS y advertencias sobre recursos mixtos o scripts locales en entorno de desarrollo.

Faro (Lighthouse)

Documentación de la pestaña Faro (Lighthouse): Ejecuta auditorías automatizadas para medir el rendimiento web, accesibilidad, optimización SEO y mejores prácticas de la plataforma legal.

Grabadora

Documentación de la pestaña Grabadora (Recorder): Permite registrar, reproducir y medir flujos de usuario automatizados paso a paso interactuando directamente con la interfaz de consulta legal.
